# 10 个 Prompt 框架（速查）

目标：把同一主题写成 10 条可复制使用的 Prompt（每条对应一个框架），并保持字段清晰、约束完整、低歧义。

## 1) ICIO 框架

- Instruction（指令）：要模型做什么（产出物 + 范围）。
- Context（背景）：场景/受众/业务目标/限制条件。
- Input Data（输入数据）：已知材料、事实、要点、样例、关键词。
- Output Indicator（输出引导）：格式、语气、长度、结构、验收标准。

## 2) CRISPE 框架

- Capacity and Role（能力/角色）：模型扮演谁（以及擅长什么）。
- Insight（洞察）：幕后信息/上下文/关键假设/潜台词。
- Statement（声明）：一句话说清任务（要产出什么）。
- Personality（个性）：输出风格（直接/严谨/幽默/克制等）。
- Experiment（实验）：要求给多个方案/版本/对比（如 3 个版本 + 说明差异）。

## 3) BROKE 框架

- Background（背景）：场景、上下文、限制条件。
- Role（角色）：模型扮演的身份。
- Objectives（目标）：要达成的目标（可量化更好）。
- Key Result（关键结果）：成功标准/验收点（结构、要点数、格式等）。
- Evolve（改进）：给出优化回路（如“如果我说不满意，用 a/b/c 三种方式迭代”）。

## 4) CREATE 框架

- Clarity（清晰度）：明确任务与边界（做什么/不做什么）。
- Relevant info（相关信息）：关键词、事实、受众、语气、格式、结构。
- Examples（实例）：给 1-2 个小例子/参考（或让模型先生成示例）。
- Avoid ambiguity（避免歧义）：把抽象要求改成可执行规则。
- Tinker（迭代）：允许/要求多轮打磨（如先给草稿→我反馈→再改终稿）。

## 5) TAG 框架

- Task（任务）：要完成的具体工作。
- Action（行动）：步骤/方法/输出结构（怎么做）。
- Goal（目标）：最终目的/效果（为什么做）。

## 6) RTF 框架

- 角色：模型扮演的角色。
- 任务：要完成的具体任务。
- 格式：输出的格式/长度/结构/语言。

## 7) ROSES 框架

- Role（角色）：模型扮演的角色。
- Objective（目标）：要实现的目标。
- Scenario（场景）：发生在什么场景/约束。
- Solution（解决方案）：期望的结果是什么。
- Steps（步骤）：为达成结果应采取的行动/流程。

## 8) APE 框架

- Action（行动）：要做什么。
- Purpose（目的）：为什么做/目标是什么。
- Expectation（期望）：期望产出/验收标准/格式。

## 9) RACE 框架

- 背景：上下文/场景/限制。
- 行动：要做什么/怎么做。
- 结果：希望得到什么结果（可验证）。
- 示例：给一个示例（或要求模型自带示例）。

## 10) TRACE 框架

- Task（任务）：要完成的任务。
- Request（请求）：对模型的明确请求（必须/禁止/约束）。
- Action（行动）：执行步骤/产出结构。
- Context（语境）：背景/受众/使用场景。
- Example（示例）：示例/参考/输出样例。
