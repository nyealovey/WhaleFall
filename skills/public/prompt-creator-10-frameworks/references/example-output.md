# 示例输出（可对照格式）

示例主题：以“职场时间管理的 3 个可落地方法”为主题，写一篇公众号短文（面向初入职场读者，300 字以内，语气直接不鸡汤，结构：标题 + 3 点要点 + 结尾 1 句行动建议）。

ICIO框架
    • Instruction 指令：写一篇公众号短文，主题为“职场时间管理的 3 个可落地方法”
    • Context 背景信息：受众为初入职场读者；语气直接、不鸡汤；避免空泛口号
    • Input Data 输入数据：需要给出 3 个可执行的方法；每个方法 1 句话说明 + 1 个小例子
    • Output Indicator 输出引导：300 字以内；输出结构为“标题 + 3 点要点 + 结尾 1 句行动建议”；中文
    Prompt：你是一名微信公众号文章作者。请为初入职场读者写一篇 300 字以内的短文，主题为“职场时间管理的 3 个可落地方法”。要求：语气直接、不鸡汤、避免空泛口号；必须给出 3 个可执行的方法，每个方法用“1 句话说明 + 1 个小例子”表达；输出结构严格为“标题 + 3 点要点 + 结尾 1 句行动建议”，只输出正文内容。

    • Personality 个性：直接、克制、少形容词，多动词；不鸡汤
    • Experiment 实验：给 2 个不同风格标题（理性/冲突感），正文保留同一套结构
    Prompt：你是“资深内容策划 + 职场效率教练”。读者是初入职场的人，痛点是事情多、频繁被打断、下班更累，他们需要可执行动作而不是理念。请写一篇公众号短文：主题“职场时间管理的 3 个可落地方法”。风格要求：直接、克制、少形容词、多动词，不鸡汤。输出要求：300 字以内；先给出 2 个标题备选（一个偏理性、一个偏冲突感），再输出正文；正文结构严格为“3 点要点 + 结尾 1 句行动建议”；每个要点必须包含“1 句话说明 + 1 个小例子”。只输出内容。

    • Key Result 关键结果：300 字以内；3 个方法每个都包含动作动词与触发条件；结尾给 1 个“今天就做”的行动指令
    • Evolve 试验并改进：如果我说“太泛/太长/太像鸡汤”，分别用 a)补充具体动作 b)压缩字数 c)重写语气 迭代一版
    Prompt：你是一名擅长方法论的公众号作者。背景：读者用碎片时间阅读，需要短、密度高、能立刻照做。目标：让读者读完就能当天试 1 个动作，降低拖延与被动加班。请写一篇 300 字以内短文，主题“职场时间管理的 3 个可落地方法”。关键结果要求：3 个方法每个都必须包含“动作动词 + 触发条件 + 1 个小例子”；结尾给 1 个“今天就做”的行动指令。若我反馈“太泛/太长/太像鸡汤”，请分别按 a)补具体动作 b)压缩字数 c)重写语气 迭代一版。

    • Examples 实例：每个方法给 1 个“办公场景”小例子（会议/消息打断/临时任务）
    • Avoid ambiguity 避免含糊不清：把“更高效”改成可执行规则（如“每天 2 次集中处理消息，各 15 分钟”）
