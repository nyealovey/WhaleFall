# React 新旧页面线上一致性复核

> 本报告只记录浏览器复核结果。本轮没有提交任何业务写操作，没有确认删除、同步、刷新、测试连接、导出、保存等会改变后端状态的动作。登录凭据未写入仓库或报告。

## 本轮结论

2026-06-25 线上重新部署后，React 新版大部分主页面的数据结构、表格列、分页和核心指标已经接近旧版。实例详情最新部署已修复“多个 tab 内容同时堆叠”的问题：当前只显示选中的 tab，未选中面板为 `hidden`。

本轮仍发现若干旧版等价差异，主要集中在全局头部、部分详情/弹窗内容和群集相关交互。以下差异已在代码侧修复，状态为“待用户重新构建部署后线上复核”：

| 优先级 | 范围 | 结论 |
| --- | --- | --- |
| P1 | 全局 Shell | 已补“修改密码”入口，链接到 `/auth/change-password`。 |
| P1 | 群集管理 | 已补 `AG账户` 查看弹窗；打开后展示 AG 总数、Contained、已配凭据、启用采集、AG 列表，并保留“同步 AG 账户”动作。 |
| P1 | 群集管理 | 已调整 `查看AG状态`，按 v1 看板返回展示副本状态表和数据库同步状态表。 |
| P1 | 定时任务 | 已把编辑任务弹窗恢复为任务名称、执行函数、秒/分/时/日/月/周/年字段。 |
| P1 | 日志中心 | 已补日志详情复制消息、复制 JSON、复制堆栈、展开上下文、复制详情入口。 |
| P1 | 凭据管理 | 已把“数据库类型”改为仅数据库凭据类型时显示；非数据库凭据不再显示。 |

## 复核范围与方法

### 复核范围

- React：`https://dbinfo.whalefall.local/console/**`
- 旧版：同域名下对应 Flask 页面
- 额外详情样本：实例 `105`，即 `jt-srmmysql-04l`
- 设置模块：告警设置、风险规则、JumpServer、Veeam、AD 设置
- 弹窗/子面板：新增、编辑、详情、权限、变更、导入、删除确认、群集 AG、任务、日志、凭据、标签、分区等入口

### 线上资源

本轮实例详情复核时生产页面加载资源：

- JS：`/console/assets/index-h7ICsT7D.js`
- CSS：`/console/assets/index-cCaEsu4U.css`
- 字体：`/console/static/css/fonts.css`

### 操作边界

- 已打开/关闭只读详情弹窗和表单弹窗。
- 删除类只打开确认弹窗，未点击确认。
- 未点击同步、刷新、测试连接、导出、保存、立即执行、暂停/恢复等会产生后端状态变化的按钮。
- 动态计数、日志数量和任务运行状态只比较结构与口径，不比较秒级变化。

3. 日志中心：补旧版日志详情中的复制消息、复制 JSON、复制堆栈、上下文展开和复制详情能力。
4. 全局 Shell：补“修改密码”入口。
5. 实例详情和列表页：删除残留英文 eyebrow/说明文字；连接状态统一中文业务文本。
