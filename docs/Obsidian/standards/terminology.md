# 术语与用词标准

## 目的

- 统一中文用词与英文术语，避免“同一个概念多种叫法”导致沟通与检索成本上升。
- 明确哪些词必须中文化，哪些属于专有名词应保留英文。

## 适用范围

- 研发：Docstring/注释、日志事件名、UI 文案、错误消息与帮助提示。
- 文档：`docs/` 下所有文档，尤其是标准/参考手册/变更记录。

## 规则（MUST/SHOULD/MAY）

### 1) 中文用词统一

- MUST：优先使用本文件给出的中文翻译（例如必须使用“账户”，避免“帐号”）。
- MUST：同一页面/同一文档内不得混用同义词（例如“实例/节点/主机”混用）。
- SHOULD：当一个概念在模型/接口中已定名（例如 `Instance`、`Tag`），文档中应与代码保持同一语义映射。

### 2) 专有名词保留英文

- MUST：框架/库/协议/产品名保持原文（例如 Flask、SQLAlchemy、Redis、Nginx）。
- MAY：首次出现时补充中文解释（便于非研发角色理解）。

### 3) 新增术语流程

- MUST：新增领域概念或面向用户的新文案前，先在本文件追加术语条目（含中文翻译与说明）。
- SHOULD：术语条目优先覆盖“领域名词 + UI 可见名词 + 日志/监控常用词”，避免无限扩张。

## 正反例

### 正例

- “账户同步会话”“实例容量统计”“标签批量分配”

### 反例

- “Account 同步 service”“同步帐号数据”（混用中英文且“帐号”不统一）

## 术语表（精选）

> 说明：术语应来自代码与产品界面；如发现未使用或歧义条目，应在评审中删除或调整。

### 核心架构术语

| 英文术语 | 中文翻译 | 说明 |
| --- | --- | --- |
| Service | 服务 | 业务逻辑服务类 |
| Manager | 管理器 | 资源/清单管理类 |
| Coordinator | 协调器 | 协调多个服务的流程编排 |
| Blueprint | 蓝图 | Flask 路由蓝图 |
| Adapter | 适配器 | 外部系统/协议适配层 |
| Wrapper | 包装器 | 对第三方组件或复杂流程的统一封装 |

### 业务领域术语

| 英文术语 | 中文翻译 | 说明 |
| --- | --- | --- |
| Instance | 实例 | 数据库实例 |
| Account | 账户 | 数据库账户 |
| Credential | 凭据 | 登录/连接凭据 |
| Permission | 权限 | 账户权限或权限快照 |
| Classification | 分类 | 账户分类（规则引擎） |

| Tag | 标签 | 资源标签 |
| Session | 会话 | 同步会话/任务会话 |

### 任务与调度术语

| 英文术语 | 中文翻译 | 说明 |
| --- | --- | --- |
| Task | 任务 | 后台任务或异步执行单元 |
| Job | 作业 | 调度器中的任务实例 |

### 常用工程术语

| 英文术语 | 中文翻译 | 说明 |
| --- | --- | --- |
| Fallback | 回退/降级 | 失败时的兜底策略 |
| Migration | 迁移 | 结构/字段/契约的版本迁移 |

## 状态用词（MUST）

> 目标：同一语义状态必须使用同一组中文展示，避免用户推断出不存在的差异（例如“禁用/停用”“进行中/执行中/运行中”混用）。

| 语义域 | 值/场景 | 中文用词（canonical） | 备注 |
| --- | --- | --- | --- |
| 资源启用态 | `is_active=true/false` | 启用 / 停用 | 仅用于“启用/停用”语义，避免用“禁用”表达资源状态 |
| 子资源存在态 | `is_active=true/false` | 正常 / 已删除 | 适用于 InstanceAccount/InstanceDatabase 等“仍存在/已删除”语义 |
| 回收站/软删 | `deleted_at!=null` | 已删除 | 对照词优先使用“正常”（不要用“在线”表达非连接语义） |
| 锁定态 | `is_locked=true/false` | 已锁定 / 正常 | 锁定与启用/停用为不同维度 |
| 运行态(生命周期) | `pending` | 等待中 | 避免 `排队中/等待/等待中` 混用 |
| 运行态(生命周期) | `running` | 运行中 | 避免 `进行中/执行中/运行中` 混用 |
| 运行态(生命周期) | `paused` | 已暂停 | |
| 运行态(生命周期) | `completed` | 已完成 | |
| 运行态(生命周期) | `failed` | 失败 | |
| 运行态(生命周期) | `cancelled` | 已取消 | |
| Scheduler 动作 | pause/resume | 暂停 / 恢复 | 避免用“启用/禁用”表达 pause/resume |
| 健康状态 | healthy/warning/unhealthy/maintenance/unknown | 正常 / 告警 / 异常 / 维护中 / 未知 | `error` 统一按“异常”展示 |

> 展示对齐：状态词可为 1/2/3 个字，但表格列/列表列等“垂直扫描”场景**必须通过样式**保证视觉对齐（推荐：为状态列设置固定列宽，并让该列内 `.status-pill` 使用 `width: 100%` + `justify-content: center`；避免使用全局 `min-width` 撑开表格导致溢出）；不要通过补空格解决。

## 门禁/检查方式

- 评审检查：UI 文案/日志事件名/Docstring 是否使用统一术语，是否出现中英文混用。
- 变更约束：新增术语时必须同步更新本文件，并在 PR 描述中说明新增原因与适用范围。
